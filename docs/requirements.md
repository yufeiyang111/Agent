## 1. 项目概述

### 1.1 项目定位

构建一个面向 **海量数据 + 高并发** 场景的 B2C 电商购物平台，支持 PC 端和移动端 Web 访问。系统需具备处理 **千万级商品 SKU、亿级注册用户、十亿级订单** 的能力，并在大促秒杀等高并发场景下保持稳定。

### 1.2 核心业务指标

| 指标 | 目标值 |
|------|--------|
| 注册用户数 | ≥ 1 亿 |
| 在线商品 SKU | ≥ 1000 万 |
| 日订单量 | ≥ 500 万 |
| 秒杀峰值 QPS | ≥ 10 万 |
| 系统可用性 | ≥ 99.99%（全年停机 ≤ 52 分钟） |
| 页面加载时间 | PC ≤ 1.5s，移动端 ≤ 2.5s（首屏） |
| API 响应时间 | 95% 的请求 ≤ 200ms |

### 1.3 技术栈概览

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | Vue 3 + Composition API + TypeScript |
| UI 组件库 | Ant Design Vue 4.x |
| 状态管理 | Pinia |
| 路由 | Vue Router 4 |
| 构建工具 | Vite 5 |
| 响应式方案 | 移动端 `viewport` + 桌面端弹性布局 |
| 后端架构 | Spring Cloud 微服务（Spring Boot 3 + Spring Cloud 2023） |
| API 网关 | Spring Cloud Gateway |
| RPC 框架 | OpenFeign |
| 数据库 | MySQL 8.0 + ShardingSphere-JDBC |
| 缓存 | Redis Cluster |
| 搜索引擎 | Elasticsearch 8.x |
| 消息队列 | Apache RocketMQ |
| 冷存储 | ClickHouse + TiDB |
| 分库分表 | Apache ShardingSphere 5.x |
| 分布式 ID | 雪花算法（Snowflake） |
| 限流降级 | Sentinel / Resilience4j |

## 5. 模块二：商品管理

### 5.1 SPU / SKU 模型

采用电商标准 SPU（Standard Product Unit）+ SKU（Stock Keeping Unit）双层模型：

```
SPU（商品） — 如 "iPhone 15 Pro Max"
  ├── 通用属性：品牌、名称、描述、图片
  ├── 销售属性模板：颜色、存储容量、网络制式
  └── SKU（具体规格） — "iPhone 15 Pro Max / 钛金属原色 / 256GB / 5G"
        ├── 价格、库存、SKU 图、SKU 编码
        └── 最终售卖单元
```

### 5.2 商品发布

| 功能项 | 详细描述 |
|--------|----------|
| 基础信息 | 商品名称、副标题、品牌、分类（三级类目） |
| 图片管理 | 主图（多张，支持拖拽排序）、详情图、视频（可选） |
| 销售属性 | 颜色、尺寸、容量等自定义属性组合生成 SKU |
| SKU 录入 | 批量录入每个 SKU 的售价、划线价、库存、SKU编码、SKU图 |
| 详情描述 | 富文本编辑器 + 模板片段（参数表格、购买须知、售后说明） |
| 标签系统 | 新品、热销、推荐、定制等标签管理 |
| 发布流程 | 草稿 → 提交审核 → 审核通过 → 上架 / 审核驳回 → 修改 |

### 5.3 商品分类

| 功能项 | 详细描述 |
|--------|----------|
| 分类层级 | 支持三级分类（一级 > 二级 > 三级），树形结构 |
| 分类属性 | 每个分类可绑定自定义属性模板（如手机分类有"电池容量""屏幕尺寸"等） |
| 分类展示 | 首页分类导航 → 二级分类 → 三级分类 → 商品列表 |
| 分类管理 | 后台支持拖拽排序、启用/禁用、设置分类 Banner 图 |

### 5.4 品牌管理

| 功能项 | 详细描述 |
|--------|----------|
| 品牌信息 | 品牌名称、Logo、品牌故事、品牌产地 |
| 品牌关联 | 品牌关联到具体分类，如"Apple"关联到"手机/平板/笔记本" |
| 品牌馆 | 品牌聚合页，展示该品牌下所有商品 |

### 5.5 商品上下架

| 功能项 | 详细描述 |
|--------|----------|
| 单商品操作 | 单个商品上架/下架 |
| 批量操作 | 多选商品批量上架/下架 |
| 定时上架 | 设置定时上架时间，到点自动上架 |
| 库存驱动下架 | SKU 库存为 0 时自动标识"售罄"，全部 SKU 售罄自动下架 |

## 7. 模块四：购物车

### 7.1 购物车功能

| 功能项 | 详细描述 |
|--------|----------|
| 加入购物车 | 选择 SKU 和数量 → 加入购物车（微动画反馈） |
| 数量修改 | 步进器增减数量，最大不能超过库存上限 |
| 商品删除 | 单条删除 / 批量删除 / 清空失效商品 |
| 选中/取消 | 单选、全选/全不选，实时计算选中商品总价 |
| 购物车列表 | 按店铺分组（平台模式）、显示商品图/名称/规格/单价/数量/小计 |
| 失效商品 | 已下架/已删除商品标灰显示"失效商品"，提示移出 |
| 库存校验 | 每次进入购物车自动校验商品库存，库存不足时提示修改数量 |
| 跨设备同步 | 未登录 → 存本地/LocalStorage；登录后 → 合并到服务端 Redis |

### 7.2 购物车业务规则

- 购物车最大容量：200 件商品（防止滥用）
- 加入购物车时设置 Redis 过期时间（7 天无操作自动清理）
- 登录时执行购物车合并策略：本地购物车 + 服务端购物车 = 去重合并
- 下单后自动从购物车移除已购商品

## 8. 模块五：订单系统

### 8.1 订单流程

```
购物车结算 / 立即购买
       │
       ▼
   确认订单页（选择地址、优惠券、发票）
       │
       ▼
   提交订单（预扣库存）
       │
       ▼
   ┌───┴───┐
   │ 待支付 │ ←─ 超时未支付 → 自动取消（恢复库存）
   └───┬───┘
       │ 支付成功
       ▼
   ┌───┴───┐
   │ 已支付 │ → 等待商家发货
   └───┬───┘
       │ 发货
       ▼
   ┌───┴────┐
   │ 已发货  │ → 物流跟踪
   └───┬────┘
       │ 确认收货
       ▼
   ┌───┴────┐
   │ 已完成  │ → 可评价、可申请售后
   └────────┘

另外的分支：
  - 待支付 → 用户主动取消 → 已关闭
  - 已支付 → 申请退款 → 处理中 → 已退款
  - 已发货 → 申请退货 → 处理中 → 已退款
```

### 8.2 订单功能

| 功能项 | 详细描述 |
|--------|----------|
| 创建订单 | 从购物车或立即购买创建，一次可包含多个商品 |
| 订单状态 | 待支付 / 已支付 / 已发货 / 已收货 / 已完成 / 已取消 / 已退款 |
| 超时取消 | 待支付订单 30 分钟未支付自动取消（RocketMQ 延迟消息实现） |
| 确认收货 | 用户可以手动确认收货，发货后 15 天自动确认收货 |
| 订单拆分 | 按店铺 / 仓库拆分订单，一个订单拆为多个子订单 |
| 订单列表 | 按状态筛选（全部/待支付/待发货/待收货/已完成/已取消） |
| 订单详情 | 订单号、商品信息（快照）、价格明细、物流信息、操作按钮 |
| 取消订单 | 待支付/待发货状态可取消，取消后恢复库存 |
| 退款/售后 | 选择原因 → 提交申请 → 商家审核 → 退款原路返回 |
| 发票管理 | 电子发票 / 纸质发票，下单时可选择是否开票 |

### 8.3 订单数据设计要点

- **分库分表**：订单表按买家 `user_id` 取模分片，8 个库 × 64 张表 = 512 张物理表
- **订单号**：基于雪花算法生成 18 位分布式 ID（时间戳 + 机器 ID + 序列号）
- **订单号 → 用户 ID 映射**：建立全局映射表，支持通过订单号查到分片键
- **商品快照**：下单时记录商品名称、图片、价格、规格快照，防止商品信息变更影响历史订单
- **冷热分离**：6 个月前的订单自动迁移至冷库（ClickHouse）

## 9. 模块六：支付系统

### 9.1 支付功能

| 功能项 | 详细描述 |
|--------|----------|
| 微信支付 | 扫码支付（PC）、JSAPI 支付（移动端 H5）、小程序支付 |
| 支付宝支付 | 扫码支付（PC）、跳转 H5 支付（移动端） |
| 余额支付 | 平台虚拟账户余额支付（需先充值） |
| 组合支付 | 余额 + 第三方支付组合支付 |
| 支付超时 | 二维码有效期 5 分钟，超时自动失效 |
| 支付结果 | 前端轮询 + 后端 Webhook 异步通知（双重确认） |

### 9.2 支付安全

- **幂等性**：支付回调使用 `payment_id` + `transaction_id` 做幂等校验
- **签名验证**：所有支付回调验证第三方签名
- **金额校验**：支付回调金额与订单金额严格比对（误差 0 容忍）
- **防重复**：Redis 分布式锁防止并发回调导致重复入账

### 9.3 退款流程

| 功能项 | 详细描述 |
|--------|----------|
| 申请退款 | 待支付 → 可直接取消退款；已支付未发货 → 全额退款 |
| 退货退款 | 已收货 → 申请退货退款 → 商家审核 → 用户寄回 → 商家确认 → 退款 |
| 退款原因 | 商品质量问题、发错货、七天无理由、不想要了等 |
| 退款到账 | 原路返回（微信/支付宝 1-7 个工作日，余额即时到账） |

## 10. 模块七：库存系统

### 10.1 库存模型

采用 **双层库存模型**：

| 库存类型 | 含义 | 说明 |
|----------|------|------|
| **总库存** | SKU 物理库存总量 | 入库时增加 |
| **预扣库存** | 已下单但未支付占用的库存 | 下单时预扣，支付时确认扣减 |
| **可用库存** | 实际可售卖的库存 | = 总库存 - 预扣库存 |
| **安全库存** | 保留的最低库存水位 | 库存低于此值时触发预警 |

### 10.2 库存扣减流程

```
下单成功 → 预扣库存 + 1（Redis LUA 脚本原子操作）
    │
    ├── 30分钟内支付成功 → 确认扣减（Redis → MySQL）
    │       └── 恢复预扣库存 → 扣减总库存 → 写入库存流水
    │
    └── 30分钟超时未支付 → 释放预扣库存（Redis LUA 脚本）
            └── 发送 MQ 消息 → 记录库存流水
```

### 10.3 库存功能

| 功能项 | 详细描述 |
|--------|----------|
| 库存扣减 | Redis 预扣 → MySQL 最终确认（两阶段提交） |
| 库存回滚 | 订单取消/超时时自动释放预扣库存 |
| 库存预警 | 库存低于安全水位时通知运营补货 |
| 库存流水 | 每次库存变动写入流水表（记录变动类型、数量、时间、关联订单号） |
| 大盘库存 | 总库存/已售/预扣/可用四维展示 |

### 10.4 库存一致性方案

- Redis 预扣使用 LUA 脚本保证原子性
- 最终一致性：MQ 异步确认扣减，失败后定时任务对账补偿
- 每日凌晨对账：MySQL 实际库存 vs Redis 缓存库存 vs 订单数据，自动修复

### 13.2 秒杀活动

秒杀是电商系统中**并发最高**的场景，需要独立设计：

| 功能项 | 详细描述 |
|--------|----------|
| 秒杀活动创建 | 设置活动时间、秒杀商品、秒杀价、秒杀库存 |
| 秒杀商品 | 独立于普通商品，单独维护秒杀库存 |
| 页面展示 | 秒杀倒计时、进度条（已抢 XX%）、动态滚动条展示最新秒杀成功记录 |
| 抢购流程 | 秒杀开始 → 点击抢购 → 排队 → 结果通知（成功/失败） |
| 限购规则 | 每个用户/每场活动限购 X 件 |

**秒杀技术方案**：

```
用户点击抢购 → Nginx 限流（单 IP 每秒 N 次）
       │
       ▼
   Sentinel 限流（秒杀接口独立限流，最大 QPS 阈值）
       │
       ▼
   RocketMQ 削峰（请求先入队列，异步消费）
       │
       ▼
   Redis LUA 校验库存 + 校验限购 → 扣库存
       │
       ▼
   返回结果 → 前端轮询等待结果 / WebSocket 推送
```

## 16. 非功能性需求

### 16.1 性能需求

| 指标 | PC 端 | 移动端 H5 |
|------|-------|-----------|
| 首屏加载时间 | ≤ 1.5s | ≤ 2.5s |
| 页面可交互时间 | ≤ 2.0s | ≤ 3.0s |
| API 响应时间（P95） | ≤ 200ms | ≤ 300ms |
| 图片加载 | 懒加载 + WebP 格式 | 同 PC |
| 商品搜索响应 | ≤ 100ms | ≤ 200ms |

### 16.2 高并发需求

| 场景 | 目标 QPS | 应对措施 |
|------|----------|----------|
| 日常访问 | 5,000 | 常规缓存 + CDN |
| 大促峰值 | 50,000 | 扩容 + 限流 + 降级 |
| 秒杀瞬间 | 100,000+ | 独立秒杀系统 + MQ 削峰 + Nginx 限流 |

### 16.3 安全需求

| 安全类型 | 措施 |
|----------|------|
| 传输安全 | 全站 HTTPS，敏感字段 AES 加密传输 |
| 认证安全 | JWT Token + refresh 机制，密码 BCrypt 加密 |
| 接口防刷 | 接口限流（令牌桶）+ 图形验证码 + 设备指纹 |
| XSS 防护 | 所有用户输入内容输出时做 HTML 转义 |
| CSRF 防护 | Token 校验 + SameSite Cookie |
| SQL 注入 | 预编译 + 参数绑定 + 分库分表自动防注入 |
| 数据脱敏 | 手机号/邮箱前端脱敏展示（138****1234） |

### 16.4 可用性需求

| 指标 | 目标 |
|------|------|
| 系统可用性 SLA | 99.99%（全年 ≤ 52 分钟不可用） |
| 部署架构 | 多机房多活部署 |
| 数据库高可用 | 主从复制 + 自动故障切换 |
| 缓存高可用 | Redis Cluster 哨兵 + 集群模式 |
| 消息队列高可用 | RocketMQ 主从同步 |
| 容灾备份 | 每天全量备份 + Binlog 实时备份 |

### 16.5 兼容性需求

| 设备 | 支持范围 |
|------|----------|
| PC 浏览器 | Chrome 90+ / Firefox 90+ / Safari 14+ / Edge 90+ |
| 移动端浏览器 | iOS Safari 14+ / Android Chrome 90+ / 微信内置浏览器 |
| 屏幕尺寸 | 320px ~ 2560px 全适配 |
| 操作系统 | Windows 10+ / macOS 12+ / iOS 14+ / Android 8+ |
