## 1. 环境要求

### 1.1 硬件要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2 核 | 4 核以上 |
| 内存 | 4 GB | 8 GB 以上 |
| 磁盘 | 10 GB 空闲 | 50 GB 以上（SSD 优先） |

### 1.2 软件要求

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| MySQL | 8.0+ | 必须 8.0 及以上（使用 utf8mb4、窗口函数等特性） |
| mysql 客户端 | 对应版本 | 用于执行 SQL 脚本 |

### 1.3 权限要求

数据库用户需要以下权限：

```sql
CREATE, DROP, ALTER ON *.*        -- 创建数据库和表
INSERT, SELECT, UPDATE ON *.*     -- 操作预置数据
CREATE VIEW ON *.*                -- 创建视图
```

使用 root 用户可满足全部要求。如使用非 root 用户，请确保授予上述权限：

```sql
GRANT ALL PRIVILEGES ON `%db%`.* TO 'your_user'@'%';
GRANT ALL PRIVILEGES ON `common_db`.* TO 'your_user'@'%';
FLUSH PRIVILEGES;
```

### 1.4 字符集配置

确保 MySQL 配置文件（my.ini / my.cnf）包含以下配置：

```ini
[client]
default-character-set = utf8mb4

[mysql]
default-character-set = utf8mb4

[mysqld]
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
```

### 2.2 执行顺序

所有 SQL 脚本设计为**幂等**的（使用 `CREATE IF NOT EXISTS` + `INSERT IGNORE`），可反复执行不会报错。

正确的执行顺序如下：

```
01_databases.sql          ← 第一步：创建数据库
       │
       ├── 02_user_center.sql     ← 第二步：在 user_db_0 建表
       ├── 03_product_center.sql  ← 第二步：在 product_db_0 建表
       ├── 04_order_center.sql    ← 第二步：在 order_db_0 建表
       ├── 05_review_center.sql   ← 第二步：在 review_db_0 建表
       ├── 06_marketing_center.sql ← 第二步：在 marketing_db_0 建表
       └── 07_common_db.sql       ← 第二步：在 common_db 建表+插数据
       │
       ├── 08_shard_replicas.sql  ← 第三步：复制到其他分片
       │
       └── 09_views.sql           ← 第四步：创建视图
```

## 3. 部署方式

### 3.1 方式一：PowerShell 部署（推荐 Windows）

PowerShell 脚本支持交互式输入密码和参数传参两种方式，有完善的错误处理。

**交互式模式（推荐）：**

```powershell
cd deploy
.\deploy.ps1
```
按提示输入 root 密码即可。

**直接传参：**

```powershell
.\deploy.ps1 -Password "your_root_password"
```

**快速模式（使用 all_in_one.sql，速度更快）：**

```powershell
.\deploy.ps1 -AllInOne -Password "your_root_password"
```

### 3.2 方式二：Windows 批处理

```cmd
cd deploy
deploy.bat your_root_password
```

或直接运行后输入密码：

```cmd
deploy.bat
```

### 3.3 方式三：Linux / macOS Shell

```bash
cd deploy
chmod +x deploy.sh
./deploy.sh your_root_password
```

交互式模式：

```bash
./deploy.sh
```

### 3.4 方式四：单文件部署（最简洁）

只需携带 `all_in_one.sql` 一个文件，适合快速迁移到其他机器：

**命令行直接执行：**

```bash
# Windows
mysql -u root -proot123 < all_in_one.sql

# Linux/macOS  
mysql -u root -p < all_in_one.sql
```

**登录 MySQL 后执行：**

```sql
mysql> source /path/to/deploy/all_in_one.sql;
```

### 3.5 方式五：手动分步执行

适合需要观察每步输出的场景：

```bash
# Step 1: 创建 23 个数据库
mysql -u root -p < 01_databases.sql

# Step 2: 创建各业务域表
mysql -u root -p user_db_0 < 02_user_center.sql
mysql -u root -p product_db_0 < 03_product_center.sql
mysql -u root -p order_db_0 < 04_order_center.sql
mysql -u root -p review_db_0 < 05_review_center.sql
mysql -u root -p marketing_db_0 < 06_marketing_center.sql
mysql -u root -p common_db < 07_common_db.sql

# Step 3: 复制到全部分片
mysql -u root -p < 08_shard_replicas.sql

# Step 4: 创建视图
mysql -u root -p common_db < 09_views.sql
```

## 5. 常见问题

### Q1: 出现 "Access denied for user 'root'@'localhost'"

**原因：** 密码错误或 MySQL 服务未运行。  
**解决：**

```bash
# 检查 MySQL 服务状态
# Windows:
sc query MySQL80
# 如未运行则启动:
net start MySQL80

# Linux:
systemctl status mysqld
systemctl start mysqld
```

### Q2: 出现 "ERROR 2003 - Can't connect to MySQL server"

**原因：** MySQL 服务未启动或端口不是默认的 3306。  
**解决：** 确认 MySQL 服务已启动，如端口不同请在脚本中添加 `-P 端口号` 参数。

### Q3: 出现 "ERROR 1117 - Row size too large"

**原因：** 表中字段过多（spu/order 表字段较多）。  
**解决：** 检查 MySQL 配置中 `innodb_page_size` 是否为默认的 16384，建议使用 8.0+ 默认配置。

### Q4: 出现 "ERROR 1410 - CREATE VIEW 权限不足"

**原因：** 数据库用户没有创建视图的权限。  
**解决：**

```sql
GRANT CREATE VIEW ON common_db.* TO 'your_user'@'%';
FLUSH PRIVILEGES;
```

### Q5: 重复执行会报错吗？

**不会。** 所有脚本都使用 `CREATE TABLE IF NOT EXISTS` 和 `INSERT IGNORE`，重复执行安全。

### Q6: 只想部署某个业务域（如只部署订单表）？

直接执行对应模块的脚本：

```bash
# 只部署订单中心（需要先执行 01_databases.sql 建库）
mysql -u root -p order_db_0 < 04_order_center.sql
```

### Q7: 部署完成后磁盘空间占用？

初始状态几乎不占用空间（空表）。各表初始大小：

| 业务域 | 空表大小 | 数据增长预估 |
|--------|---------|-------------|
| 用户中心 | ~1 MB | 1 亿用户 ≈ 50 GB |
| 商品中心 | ~1.5 MB | 1000 万 SKU ≈ 20 GB |
| 订单中心 | ~2 MB | 日 500 万单 ≈ 500 GB/年 |
| 评价中心 | ~0.5 MB | 1 亿评价 ≈ 100 GB |
| 营销中心 | ~0.5 MB | 较小 |
| 公共库 | ~0.5 MB | 极小 |

## 6. 后续配置

### 6.1 ShardingSphere-JDBC 配置

数据库初始化完成后，需在应用中配置 ShardingSphere-JDBC 分片规则：

```yaml
spring:
  shardingsphere:
    datasource:
      names: user_db_0,user_db_1,user_db_2,user_db_3,
             product_db_0,product_db_1,product_db_2,product_db_3,
             order_db_0,order_db_1,order_db_2,order_db_3,
             order_db_4,order_db_5,order_db_6,order_db_7,
             review_db_0,review_db_1,review_db_2,review_db_3,
             marketing_db_0,marketing_db_1,
             common_db
      # ... 各数据源连接池配置
    rules:
      sharding:
        tables:
          t_order:
            actualDataNodes: order_db_${0..7}.order_${0..63}
            databaseStrategy:
              standard:
                shardingColumn: user_id
                shardingAlgorithmName: order_db_inline
            tableStrategy:
              standard:
                shardingColumn: user_id
                shardingAlgorithmName: order_table_inline
```

### 6.2 数据同步链路配置

```
MySQL (主库)
   ├── Canal → MQ → Elasticsearch (商品搜索)
   ├── DataX → ClickHouse (OLAP 分析)
   └── 定时任务 → 冷数据归档至 ClickHouse
```

参考 `docs/数据库说明文档-第一期.md` 第 9 节配置冷热分离定时任务。

## 7. 分片路由规则参考

| 业务域 | 表 | 分片键 | 库路由 | 表路由 |
|--------|-----|--------|--------|--------|
| user_db | user | user_id | `(user_id>>4) % 4` | `(user_id>>4) % 32` |
| user_db | user_address | user_id | 同 user 库 | 同 user 库 |
| product_db | spu | spu_id | `spu_id % 4` | `(spu_id>>4) % 32` |
| product_db | sku | spu_id | 同 spu 库 | 同 spu 库 |
| order_db | order | user_id | `user_id % 8` | `(user_id>>4) % 64` |
| order_db | order_item | user_id | 同 order 库 | 同 order 库 |
| review_db | review | spu_id | `spu_id % 4` | `(spu_id>>4) % 32` |
| marketing_db | coupon | coupon_id | `coupon_id % 2` | `(coupon_id>>4) % 16` |
| marketing_db | user_coupon | user_id | `user_id % 2` | `(user_id>>4) % 16` |
| common_db | 全部 | 不分片 | 1 个库 | 1 张表 |
